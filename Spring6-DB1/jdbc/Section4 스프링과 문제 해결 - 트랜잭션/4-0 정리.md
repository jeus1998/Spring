### 애플리케이션과 JDBC DB

```text
애플리케이션 - DB

각각의 데이터베이스마다 커넥션을 연결하는 방법, SQL을 전달하는 방법, 그리고 결과를 응답 받는 방법이 모두 다르다는 점

그래서 JDBC 등장 

애플리케이션 - JDBC - MySQL 드라이버<JDBC 드라이버> - MySQL 
애플리케이션 - JDBC - Oracle 드라이버<JDBC 드라이버> - Oracle

JDBC(Java Database Connectivity)는 자바에서 데이터베이스에 접속할 수 있도록 하는 자바 API

JDBC 한계

JDBC의 등장으로 많은 것이 편리해졌지만, 각각의 데이터베이스마다 SQL, 데이터타입 등의 일부 사용법 다르
다. ANSI SQL이라는 표준이 있기는 하지만 일반적인 부분만 공통화했기 때문에 한계가 있다

JPA(Java Persistence API)를 사용하면 이렇게 각각의 데이터베이스마다 다른 SQL을 정의해야 하는
문제도 많은 부분 해결할 수 있다
```

### JDBC 최신 데이터 접근 기술 

```text
DB 접근 여러가지 방법 

1. 애플리케이션 -> JDBC 직접 접근 -> DB 드라이버 ->  데이터베이스

2. 애플리케이션 -> SQL Mapper (JDBC Template, MyBatis) -> JDBC -> DB 드라이버 -> 데이터베이스

장점
- JDBC를 편리하게 사용하도록 도와준다.
- SQL 응답 결과를 객체로 편리하게 변환해준다.
- JDBC의 반복 코드를 제거해준다.
단점
- 개발자가 SQL을 직접 작성해야한다.

3. 애플리케이션 -> JPA (Hibernate) -> JDBC -> DB 드라이버 -> 데이터베이스

ORM은 객체를 관계형 데이터베이스 테이블과 매핑해주는 기술이다.
이 기술 덕분에 개발자는 반복적인 SQL을 직접 작성하지 않고, ORM 기술이 개발자 대신에 SQL을 동적으로 만들어 실행해준다.
추가로 각각의 데이터베이스마다 다른 SQL을 사용하는 문제도 중간에서 해결해준다.

JPA는 자바 진영의 ORM 표준 인터페이스
JPA 구현체: 하이버네이트와 이클립스 링크 등의 구현 기술
```
